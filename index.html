    API DQ3

    Authentication

    To authorize, use this code:

    const kittn = require('kittn');

let api = kittn.authorize('meowmeowmeow');

    Make sure to replace meowmeowmeow with your API key.

    Kittn uses API keys to allow access to the API. You can register a new Kittn API key at our developer portal.

    Kittn expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: meowmeowmeow

    You must replace meowmeowmeow with your personal API key.

    Space

    Request body:

    
"space":[ 
    {
        "site_id": [1234, 456], 
        "site_level2":["A", "B"]
    },
    {
        "site_id": [79]
    }
]

    Sélection des sites/Niveaux 2 choisis pour l’analyse.

    L’objet « space » est une liste d’objet qui sont composé comme suit :

    Objet Description
    site_id liste d’identifiant de site, séparés par des virgules
    site_level2 liste d’identifiant de site, séparés par des virgules

    Periodes

    L’objet « p1 » est obligatoire, il décrit la période de référence ou principale d’analyse.

    L’objet « p2 » est facultatif, il décrit la période de comparaison.

    Seules les périodes « p1 » et « p2 » fonctionnent aujourd’hui.

    Périodes absolues

    Request body

    "period": {

    "p1": [

      {

        "type": "D",

        "start": "2017-07-01",

        "end": "2017-07-01"  <Optionnel, si absent end = start>

      }

    ],

    "p2": [

          {

            "type": "D",

            "start": "2017-07-01",

            "end": "2017-07-01"  <Optionnel, si absent end = start>

          }

        ]

  }

    Format de Date : yyyy-mm-dd

    Périodes relatives

    Request body

    "period": {

    "p1": [

      {

        "type": "R",

        "granularity": "D", // Y, Q, M, W, D

        "offSet": "-1"

      }

    ],

    "p2": [

      {

        "type": "R",

        "granularity": "D", // Y, Q, M, W, D

        "offSet": "-2"

      }

    ]

  }

    L’heure actuelle dans le fuseau horaire du site le plus à l’ouest est le référentiel.

    Paramètres Description
    granularity Granularité de la période
    offset Décalage de la période

    Evolution

    Request body 

    "evo": { 

    "granularity": "D", 

    "top": { 

      "max-results": 5, 

      "page-num": 1, 

      "sort": ["-m_visits"], 

      "period": { 

        "p1": [ 

          { 

            "type": "D", 

            "start": "2017-05-12", 

            "end": "2017-05-16" 

          } 

        ] 

      } 

    } 

  }

    Le paramètre « evo » décrit le contexte du calcul de l’évolution de l’analyse. Il est composé de différents paramètres obligatoires :

    Paramètres Description Valeurs possibles
    granularity Granularité de l'évolution M(onth), W(eek), D(ay), H(our)
    top Description des éléments sélectionnés, top des éléments en fonction de la période, de la pagination et du tri, le nombre d’éléments est dépendant des paramètres Voir en dessous
    Top parameter Description Valeurs possibles
    max-results - any number
    page-num - any number

    Sort

    Request body:

    
"sort":["m_visits", "-p1.m_uv"]

    Le "-" permet de trier de manière décroissante.

    Il est possible de trier sur autant de colonnes que souhaitées.

    Il n'est pas possible de trier sur une propriété, le tri s'effectue seulement sur une ou plusieurs métriques.

    Il n'est pas possible de trier sur une colonne non présente dans le paramètre "columns" de l'URL.

    Columns

    Request body

    "columns": [ 

    "visit_src", 

    "visit_device_type", 

    "m_visits"

       "p1.m_visits"
